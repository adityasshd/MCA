# Unit 14 Network Security

## Summary

Network security encompasses various techniques to protect data and communication systems. It involves understanding cryptographic methods, identifying potential threats, and implementing strategies to maintain confidentiality, integrity, and availability. Key components include:

- **Encryption & Decryption**
- **Cryptography Types**: Symmetric, Asymmetric, and Hash Functions
- **Security Goals**: Confidentiality, Integrity, Availability
- **Security Services**: Data Protection, Identity Verification, Non-repudiation
- **Defensive Strategies**: Prevention, Protection, Resilience

Understanding these fundamentals helps in designing robust network security frameworks that safeguard digital assets effectively.

## Self Assessment
1. What is the primary difference between symmetric and asymmetric key cryptography?
2. Explain the concept of non-repudiation in network security.
3. How does a hash function contribute to data integrity?

## Answers for Self Assessment
1. **Symmetric Key Cryptography**: Uses a single shared key for both encryption and decryption.
   **Asymmetric Key Cryptography**: Uses a public key for encryption and a private key for decryption.

2. **Non-repudiation** ensures that a sender cannot deny having sent a message or transaction. It provides proof of origin and authenticity.

3. **Hash Functions** convert plaintext into a fixed-length hash value. Any change in the original data will produce a different hash, allowing verification of data integrity.
